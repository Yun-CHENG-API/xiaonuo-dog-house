# 自定义题库模板说明

每个孩子学的内容不一样，这个 App 支持上传一份 **自定义题库 JSON**，在「⚙️ 自定义题库」里粘贴或上传，替换默认的题库。没有提供的类别会继续使用默认题库。

## 比"对不对"更重要的事

这个 App 的目的不是让孩子刷题、追求正确率，而是借助番茄钟和小游戏，帮孩子养成**思考的习惯**——做完一类题之后，能自己归纳总结、联系以前学过的知识、想一想"为什么"。

所以在生成自定义题库时，请遵循两条原则：

1. **能力分类**：每道题最好能体现一种思维方式，而不只是记忆点。例如：归纳分类（这几个词有什么共同点）、因果推理（为什么飞机要逆风起飞）、类比联想（这首诗和哪首诗很像）、错因分析（这次为什么错了，和上次的错有什么共同点）。
2. **思考提示优先**：可以的话，给题目配一句开放式的"想一想"（`th` 或 `reflect` 字段），引导孩子说出自己的想法，而不是只看一个标准答案。

## 顶层结构

把所有内容放进一个 JSON 对象里，键名如下（都是可选的，缺省的类别用默认题库）：

```json
{
  "words": [ ... ],
  "air": [ ... ],
  "chinese": [ ... ],
  "poem": [ ... ],
  "order": [ ... ],
  "reinforce": [ ... ],
  "reflect": { ... }
}
```

## 1. `words` —— 英语单词配对（🔤 单词游戏）

数组，每一项是一个对象，包含英文、中文、音标，以及答对后展示的"近义/同类词""反义/对比词"和一个例句：

```json
"words": [
  {
    "en": "plane",
    "zh": "飞机",
    "ipa": "/pleɪn/",
    "syn": [["aircraft", "飞机"], ["jet", "喷气式飞机"]],
    "ant": [["boat", "船"], ["car", "汽车"]],
    "ex": { "en": "The plane took off from the airport.", "zh": "飞机从机场起飞了。" }
  },
  {
    "en": "runway",
    "zh": "跑道",
    "ipa": "/ˈrʌnweɪ/",
    "syn": [["airport", "机场"], ["track", "跑道"]],
    "ant": [["sky", "天空"], ["tower", "塔台"]],
    "ex": { "en": "The plane is moving down the runway.", "zh": "飞机正在跑道上滑行。" }
  }
]
```

字段说明：
- `en` / `zh`：英文单词和中文意思（必填）。
- `ipa`：音标，可以留空字符串 `""`，答对/答错后会显示出来。
- `syn`：2个"近义词或同类词"，格式为 `[英文, 中文]` 的数组，找不到严格近义词时可以填同类词、上下位词等关联词，让显示永远不为空。
- `ant`：2个"反义词或对比词"，格式同上，找不到严格反义词时也可以填相关对比词。
- `ex`：例句，包含 `en`（英文句子）和 `zh`（中文翻译），答完后会展示。
- `syn` / `ant` / `ex` 都是可选的，缺省则答完后不显示对应内容。

- 至少建议 15-20 个，随机抽 10 个出题。
- 朗读使用浏览器的英式男声，发音以英文单词为准。

> 兼容提示：旧版的 `[英文, 中文, 音标]` 三元组格式仍然支持，App 会自动识别并补全为新格式（`syn`/`ant` 为空数组，`ex` 为空），但建议升级为新格式，让孩子答对后能看到更多关联信息。

## 2. `air` —— 知识问答（✈️ 航校 / 可改成任意主题，比如科学、历史）

数组，每一项：

```json
{
  "ph": "🧰 起飞前",
  "q": "起飞前，机长围着飞机走一整圈，是在干什么？",
  "o": ["检查飞机外面有没有问题", "锻炼身体", "和飞机拍照留念"],
  "a": 0,
  "ex": "这叫"绕机检查"，真正的检查单第一项就是它！",
  "th": "想一想：为什么坐在驾驶舱里检查仪表还不够？"
}
```

字段说明：
- `ph`：阶段/分类标签（显示为一个小标签，可以按主题、单元划分）
- `q`：题目
- `o`：选项数组（2-4个）
- `a`：正确选项的下标（从0开始）
- `ex`：答完后展示的讲解
- `th`：**思考题**——开放式问题，鼓励孩子说出自己的想法（重点字段，建议每题都写）

这一类很适合改造成"科学小问答""历史小问答"等，关键是 `th` 字段要能引发联想和推理，不要只是"还有什么例子"这种泛泛的话。

## 3. `chinese` —— 选择题知识点（📖 语文知识点 / 也可用于其他学科）

数组，每一项：

```json
{
  "q": "\"美丽\"的近义词是？",
  "o": ["漂亮", "丑陋", "高大", "聪明"],
  "a": 0,
  "ex": "\"美丽\"和\"漂亮\"意思相近，都是形容好看、好。"
}
```

- `q`：题目
- `o`：选项数组（2-4个）
- `a`：正确选项下标
- `ex`：讲解。建议讲解里除了说明"为什么对"，也点一下**这类题的通用方法**（比如"近义词题可以先想字面意思再想感觉"），帮孩子归纳"套路"而不是死记答案。

这个题库结构很通用，换成数学、英语语法、科学概念都可以。

---

## 4. `poem` —— 古诗/古文填空（📜 古诗游戏）

数组，每一项是一句诗里挖掉一个字：

```json
{
  "title": "《绝句》· 杜甫",
  "line": "____日江山丽，春风花草香。",
  "ans": "迟",
  "py": "chí",
  "mean": "春天日子渐长，江山一片秀丽，春风吹来，花草都散发着芳香。"
}
```

- `title`：诗题和作者
- `line`：诗句，挖空处用 `____`（4个下划线）代替
- `ans`：正确的字
- `py`：该字的拼音（答完才显示）
- `mean`：（可选）这句诗的现代解释，用孩子能懂的话讲一遍意思，答完后会显示在拼音下面。不写也不会报错，只是不会显示这一行。

建议同一首诗出 2 句（挖不同的字），方便孩子读完整首诗、体会意境，而不只是死记单字。**儿童内容安全提示**：避免选择涉及死亡、自尽等沉重主题的诗句。

---

## 5. `order` —— 英语句子排序（🔤 句子排序游戏）

数组，每一项：

```json
{
  "cn": "我有一支红色的笔。",
  "words": ["I", "have", "a", "red", "pen"],
  "punc": "."
}
```

- `cn`：中文提示
- `words`：按正确顺序排列的英文单词数组
- `punc`：句末标点（`.` `?` `!`）

---

## 6. `reinforce` —— 错题强化练习（🎯 错题强化游戏）

数组，每一项：

```json
{
  "subject": "数学",
  "q": "168÷12的商是多少？",
  "o": ["14", "41", "12", "21"],
  "a": 0,
  "ex": "用竖式算一算：12×14=168 ✓。今天作业里这道题写错了答案位置，算完一定要回头检查一下哦～"
}
```

- `subject`：学科标签（数学/语文/英语……）
- 其余字段同 `chinese`
- `ex` 里最好能点出**孩子上次错在哪里、这次要注意什么**，帮助"温故知新"，不是简单换一道同类题。

这一类题库建议**持续累积**：每次批改完作业后追加新题，而不是覆盖旧题，让孩子能不断巩固容易再错的知识点。

## 7. `reflect` —— 自定义"复盘想一想"提示语（可选）

每个游戏结束后，App 会随机弹出一组"想一想"问题，让孩子用语音或文字写下自己的想法（存入「📝 复盘小本」）。可以按游戏类型自定义这些提示语：

```json
"reflect": {
  "words": [
    { "q1": "这些单词里，哪一个你今天印象最深？", "q2": "为什么是它？是因为读音、意思，还是别的原因？" },
    { "q1": "挑一个新学的单词，说一句用到它的话。", "q2": "这句话里还能换成别的单词吗？换了之后意思有什么不一样？" }
  ],
  "chinese": [
    { "q1": "这些题目里，有没有一个知识点你以前学过？", "q2": "今天再遇到它，和上次比，你多懂了什么？" }
  ]
}
```

可用的键：`words` `math` `air` `chinese` `poem` `order` `reinforce`。每类建议 2-3 组，App 会每次随机选一组。

每组提示语是 **两步走** 的设计：

- `q1`：一个具体、好回答的小问题（比如"挑一个/选一个/说说看"），帮孩子先开口，降低表达的门槛。
- `q2`：紧接着的"为什么/有什么联系"追问，引导孩子归纳总结、联系旧知识、做因果或类比推理——这才是真正锻炼思考的部分。

**这两步提示语是整个 App 最重要的部分**——`q1` 让孩子敢说、容易说，`q2` 让"说"变成"想"。避免把 `q2` 写成"你觉得难不难""开心吗"这种没有思考空间的问题。

> 兼容提示：旧版自定义题库里 `reflect` 如果是字符串数组（如 `"words": ["...", "..."]`），App 仍能识别，会把每个字符串当作 `q1`，自动补一句通用追问"为什么呢？能再多说一点吗？"作为 `q2`，但建议升级为 `{q1,q2}` 格式，写一句更贴合内容的追问，效果会更好。

## 怎么用AI生成自己的题库

1. 把孩子的课本内容、单词表、阅读的书拍照或导出文字，发给AI（比如这个对话）。
2. 告诉AI："请按照 QUESTION_BANK_TEMPLATE.md 的格式，帮我把这些内容做成题库 JSON，每道题尽量配一句能引导孩子归纳总结/联系旧知识/问为什么的思考提示。"
3. AI生成后，复制 JSON，打开 App，点右上角「⚙️」，粘贴进去，点击「保存并应用」。
4. 觉得不合适，随时点「恢复默认题库」。
